<objective>
Realize the SVC-06 recondition MAC path in WASM (D-01) and the client-side 409 hash-mismatch semantics (Open Q1): decode the OPFS tensor (11-01 reader), apply per-source conditioning through `compose_gain` + `readout_coherent`, and gate the whole thing on a re-minted blake3 tensor identity so a mismatched hash is refused with a typed `HashMismatch` — never silently served (D-12). Purpose: SVC-06 was designed in Phase 6 (server 409 gate against a stub tensor) and is realized end-to-end here. Per Open Q1 the 409 SEMANTICS run client-side (stale + refuse-to-MAC on hash mismatch); the existing `envi-service/src/api/calc.rs` recondition DTOs stay the frozen wire shape for the designed-ahead contract. Conditioning is a READOUT param excluded from tensor identity (Phase-6 D-07), so a conditioning edit never stales.
