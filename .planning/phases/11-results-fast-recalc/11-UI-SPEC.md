# Phase 11 — UI Design Contract

> Visual and interaction contract for the Results & Fast Recalc surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase is **data-visualization-heavy** (isophone maps, spectra, difference maps, object styling). Beyond the standard chrome contract it declares **validated dataviz palettes**, a **component & interaction inventory**, and the **app-wide info-button** contract. All values are prescriptive.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (established metrao3 token system, not shadcn) |
| Preset | not applicable |
| Component library | none — hand-built React components on the metrao3 token layer (`web/src/theme.css` + `web/src/app.css`) |
| Icon library | hand-authored inline SVG (`web/src/icons.ts`), 16px, 1.4 stroke, `currentColor` — offline, no icon font |
| Font | system sans (`--font-sans`) + system mono (`--font-mono`) — no web-font fetch (offline P5) |
| Theme mode | **dark-only** (metrao3 is dark-only; the dataviz "light/dark aware" requirement is satisfied by validating data palettes against the dark surface `#14171c`, not by shipping a light theme) |

**shadcn gate resolution:** A mature, deliberately-chosen design system already exists (metrao3 tokens, adopted verbatim, dark-only, offline). CLAUDE.md and the approved 07-UI-SPEC forbid inventing tokens. Initializing shadcn would fork the token system and regress the Phase-7/8/9/10 panels this phase must retrofit. **Tool: none is a deliberate decision, not an absence.** Registry safety gate: not applicable.

## Spacing Scale

Uses the **frozen metrao3 spacing tokens** (`web/src/theme.css`). Do not invent new spacing values (D-11 "do not invent tokens").

| Token | Value | Usage |
|-------|-------|-------|
| `--space-0` | 2px | Segmented-control inner gap, hairline insets |
| `--space-1` | 4px | Icon gaps, dense row padding, table cell gaps |
| `--space-2` | 8px | Compact element spacing, control gaps |
| `--space-3` | 12px | Panel-header padding, chart-to-table gap |
| `--space-4` | 16px | Default panel padding, shell gutters |
| `--space-5` | 20px | Dialog body padding |
| `--space-6` | 24px | Section breaks |
| `--space-8` | 32px | Major section breaks |
| `--space-12` | 48px | Overlay scrim padding |

**Exceptions to the 8-point rule (pre-existing, binding):** the metrao3 scale intentionally includes `2px`, `12px`, and `20px`. These are inherited verbatim and are NOT to be "corrected" to multiples of 4 — doing so would diverge from the adopted token source and the shipped Phase-7/8/9/10 UI. Touch targets: `44px` min-height on primary/destructive `.btn` (kiosk/phone), `28px` (`--row-h`) / `36px` (`--row-h-lg`) on dense desktop rows.

## Typography

Uses the **frozen metrao3 type scale** (`--text-*`). This phase uses four sizes and two weights (regular 400 + medium 500); no other weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / control text | `--text-sm` 13px | 400 | 1.4 |
| Label / caption / table cell | `--text-xs` 12px | 400 | 1.4 |
| Micro (band index, unit, chip, axis tick) | `--text-xxs` 11px | 400 | 1.4 |
| Section title / panel heading | `--text-base` 15px | 500 | 1.4 |

Notes:
- Numeric readouts (spectrum table dB values, band index, axis ticks, legend break values) use `--font-mono` with `font-variant-numeric: tabular-nums` so columns align vertically (dataviz figures rule).
- SVG chart axis labels use the existing `.curve-axis-label` (mono, 9px) idiom from `SpectrumEditor`.
- No display/serif face anywhere. Larger sizes (`--text-lg`..`--text-3xl`) exist but this phase does not introduce new uses of them.

## Color

Two distinct color systems. **UI chrome** follows metrao3 60/30/10; **data-viz color** follows the dataviz-validated palettes and is explicitly NOT drawn from the brand accent (a dataviz non-negotiable: data color ≠ brand color).

### UI chrome (60/30/10)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg` `#0b0d10`, `--color-surface` `#14171c` | App background, map slot, panel surfaces |
| Secondary (30%) | `--color-surface-2` `#1b1f26`, `--color-surface-3` `#232831`, `--color-border` `#2a3038` | Cards, right-rail panels, inputs, table headers, borders |
| Accent (10%) | `--color-primary` `#4ea8ff` | See reserved-for list below |
| Destructive | `--color-crit` `#ef4444` | Delete-scenario confirm, hash-mismatch reject banner |

**Accent (`--color-primary`) reserved for:** active/selected tool + segmented-control segment; input focus ring (`:focus-visible`); links; the primary CTA button fill (`.btn.primary`); the selected receiver row/marker highlight. **Never** used as a data-series color, an isophone class color, or a difference-map pole (those come from the validated data palettes).

**Severity tokens** (`--color-ok` `#22c55e`, `--color-warn` `#f59e0b`, `--color-crit` `#ef4444`, `--color-info` `#4ea8ff`) are reserved for state (stale badge, reject banner, weighting-mismatch warnings) and always ship with a text label or icon — never color alone.

### Data-viz palettes (validated — D-19)

Validated with `scripts/validate_palette.js` against the **metrao3 dark surface `#14171c`** on 2026-07-12. See "Data Visualization Palettes" below for the full ramps, break tables, and validator evidence.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (results) | **"Export…"** (opens the export menu: GeoTIFF / GeoJSON / CSV) |
| Scenario CTA | **"New scenario"** (clone-then-edit); save action **"Save scenario"** |
| Difference-map CTA | **"Compare scenarios"** (pick A and B) |
| Conditioning (no button — live) | Live debounced recalc; the affordance label is the source name + **"Gain / Filter / Delay"** group headers |
| Empty state — no results | Heading: **"No results yet"** · Body: **"Run a calculation from the Calc panel to see the spectrum and noise map here."** |
| Empty state — no receiver selected | Heading: **"Select a receiver"** · Body: **"Click a receiver on the map or pick one from the list to read its spectrum."** |
| Empty state — no scenarios | Heading: **"One scenario"** · Body: **"You have the base scenario only. Create a weather scenario to compare conditions."** |
| Error — stale results (hash mismatch) | Badge: **"Out of date"** · Detail: **"The scene changed since this result was computed. Recompute to refresh the noise map and spectra."** |
| Error — MAC refused (409 semantics) | **"Can't recondition — this result no longer matches the current scene. Recompute first."** |
| Error — malformed / missing tensor chunk | **"Result data is unavailable or corrupt. Recompute this scenario."** |
| Error — weighting/label mismatch | **"This result was computed as dB({X}); the label follows the result, not the toggle."** |
| Destructive confirmation — delete scenario | Title: **"Delete scenario "{name}"?"** · Body: **"This removes its cached result. The base scenario and other scenarios are unaffected."** · Confirm button: **"Delete scenario"** (`.btn.danger`) |
| Export attribution footer (every export) | **"Generated by ENVI · Nord2000 (AV 1106/07) · CRS {epsg} · {dB weighting} · data © OpenStreetMap / Overture / ESA WorldCover / Copernicus"** |

Copy rules: English-only; sentence case for body, controls, and buttons; units always shown (dB, dB(A), dB(C), Hz, m, ms); band identity shown as **band index + exact Hz** (never nominal Hz alone). Never paste AV 1106/07 text (copyright — cite by report number).

## Data Visualization Palettes

> Loaded the `dataviz` skill (D-19). Every palette below was run through `scripts/validate_palette.js` against surface `#14171c` (dark). Data color is a separate system from the UI accent. Color is **never load-bearing alone**: every scale ships legend + numeric break labels, and area objects add hatch (texture) — the dataviz relief rule for the 8–12 CVD floor band.

### 1. Isophone map — SEQUENTIAL (magnitude, D-03/D-04)

Two preset families; the scale is `presets + editable breaks` (D-04), and **legend breaks ≡ contour breaks ≡ class colors is enforced** (a single source-of-truth `breaks[]` + `colors[]` array drives the contour tracer, the MapLibre fill layer, and the legend — never three parallel definitions). Editing breaks re-runs only the iso-band tracer over the cached level grid (SC3 — no re-solve).

**Default preset — "EU END" fixed ~5 dB bands** (D-03, familiar to acousticians). Green → yellow → orange → red → violet, ascending. Default break edges (dB, weighting from result metadata):

| Band | Range (dB) | Fill (hex) | Note |
|------|-----------|-----------|------|
| 1 | < 55 | `#7fbf7f` (green) | below-lowest, lightest |
| 2 | 55–60 | `#bfdc6b` (yellow-green) | |
| 3 | 60–65 | `#f6e04b` (yellow) | |
| 4 | 65–70 | `#f4a637` (orange) | |
| 5 | 70–75 | `#e34948` (red) | |
| 6 | ≥ 75 | `#8a2be2` (violet) | above-highest, darkest |

> ⚠ **A4 (RESEARCH Assumptions Log):** these EU END band edges and hex are the *canonical-style* default; confirm the exact END/CNOSSOS Lden band edges + official hex with the user or the END Annex before ship. The END ramp is a rainbow and is NOT perceptually uniform — this is accepted *only* because it is the domain-standard familiar default AND is always accompanied by numeric break labels + the table view (color is never the sole encoding). Do not use the END ramp for any non-map encoding.

**Perceptually-uniform presets (D-04, offered alongside END):** `viridis` and `turbo`, sampled at the same break count. These are the correct default when the user is doing quantitative reading rather than END-comparison. Both are single-progression, monotonic in lightness (viridis) / designed for detail (turbo). Ship canonical stops; sample N colors for N breaks.

**Editable-breaks contract (mirrors NoizCalc TI 386 §4.6.5):** smallest-interval value, interval magnitude (dB), number of intervals, ascending toggle, keep-color-sequence toggle; interval sizes need not be constant (editable upper limits). Validation (V5): breaks strictly monotonic increasing, finite, ≥ 2 breaks; reject non-monotonic in WASM with a typed error surfaced as an inline `.form-error`.

### 2. Difference map — DIVERGING (polarity, D-16)

**blue ↔ red, neutral gray midpoint at 0** (dataviz diverging pair). Per-receiver dB(A) total delta (scenario A − B), symmetric clamp about 0, equal steps per arm, gray at the midpoint (never a hue at 0).

| Role | Dark-surface hex |
|------|------------------|
| Negative pole (A quieter) | `#2a78d6` (blue) |
| Mid arm − | `#7aa8e0` |
| Neutral midpoint (Δ ≈ 0) | `#383835` (gray, dataviz dark neutral) |
| Mid arm + | `#e08a8a` |
| Positive pole (A louder) | `#d03b3b` (red) |

Midpoint/clamp is Claude's discretion (CONTEXT): default **symmetric clamp at ±max(|Δ|)** rounded to the nearest 1 dB, with a dead-zone of ±0.5 dB rendered as neutral (below just-noticeable difference). Legend shows signed dB values; a table view lists per-receiver Δ. The brand accent `#4ea8ff` is deliberately NOT a pole (avoid confusing UI-selection blue with data-negative blue).

### 3. Scene objects — CATEGORICAL (identity, D-17/D-18/D-19)

The 9 frozen kinds (`kinds.ts`). Color carries identity **plus** a geometry-specific secondary channel — this is what lets objects sit at full styling on top of the isophone fill (D-18). The categorical hues are the dataviz dark 8-slot set (below); color may repeat **across** geometry types but is unique **within** a geometry type (the existing `kinds.ts` reuse principle, extended). Symbol/width/hatch is the load-bearing channel per NoizCalc TI 386 §4.6.3.

**Validator evidence (2026-07-12, surface `#14171c`, dark):** 8-slot set `#3987e5,#199e70,#c98500,#008300,#9085e9,#e66767,#d55181,#d95926` — Lightness band PASS, Chroma floor PASS, **CVD separation WARN** (worst adjacent green↔yellow ΔE 10.3 protan — the 8–12 floor band), Contrast vs surface PASS. Floor band is **legal here because hatch/symbol + legend are the mandated secondary encoding** (dataviz relief rule).

Per-kind styling (NoizCalc §4.6.3 model — point = symbol/size/border, line = width/color, area = fill + border + **separate hatch**):

| Kind | Geometry | Color (dark) | Secondary channel |
|------|----------|--------------|-------------------|
| source | point | `#3987e5` blue | `source` glyph, filled marker + border |
| receiver | point | `#199e70` aqua | `receiver` glyph; selected → accent ring |
| elevation_point | point | `#c98500` yellow | `elevation_point` diamond glyph |
| wall | line | `#e66767` red | 3px solid line |
| elevation_line | line | `#c98500` yellow | 2px dashed line |
| building | area | `#9085e9` violet | fill 45% + 1px border + **hatch: solid-diagonal 45°** |
| forest | area | `#008300` green | fill 35% + border + **hatch: dotted / stipple** |
| ground_zone | area | `#d95926` orange | fill 30% + border + **hatch: cross-hatch** |
| calc_area | area | `#d55181` magenta | fill 15% + dashed border + **hatch: sparse 135° lines** (frame emphasis, low fill) |

Hatch patterns are runtime-generated on a `<canvas>` → `map.addImage(id, imageData)` → `fill-pattern` (MapLibre has no native hatch). Angles from the dataviz texture rule (45° / 135°). Semi-transparent fills (§4.6.5) let the hatched area read over the isophone fill. Draw order (§4.6.3): isophone fill (bottom) → areas → lines → points (top). **⚠ D-19/Pitfall 8:** this restyles the *display* layers in `web/src/map/` only — it must NOT change Terra Draw's *draw-time* styling/validation; regression-test the Phase-7 editor journeys.

### 4. Spectrum chart — series colors (D-05/D-08)

Inline SVG (matches `SpectrumEditor`, zero new JS dep), dataviz-styled. Series:

| Series | Color | Mark |
|--------|-------|------|
| Total level (primary) | `#3987e5` blue (categorical slot 1) | bars (per-band) or 2px line |
| Coherent (overlay, on-demand) | `#199e70` aqua (slot 2) | 2px line |
| Incoherent (overlay, on-demand) | `#d95926` orange (slot 8) | 2px line, distinct from coherent |

A ≥2-series overlay always shows a legend + direct labels (dataviz). Grid/axes recessive (`--color-border` / `--color-text-dim`). Bars: 4px rounded data-end anchored to baseline, 2px surface gap between adjacent bars (dataviz mark spec). Hover: per-band tooltip (band index, exact Hz, dB total + coherent/incoherent split). dB(A)/dB(C) toggle re-labels the axis instantly (precomputed tables, no recompute — D-09).

## Component & Interaction Inventory

New Phase-11 surfaces (right-rail panels + map overlays + dialogs), on the existing four-region shell (`app.css`).

### Spectrum Panel (`SpectrumPanel.tsx`, WEB-11)
- **Chart primary + expandable exact-numbers table** (D-05). Chart = inline SVG bar/line per-band + totals; table (`.spectrum-table` idiom, reused) = expert readout/copy with columns **band index · exact Hz · dB**.
- **Display toggle** (segmented `.switch`): **1/3-oct (default)** ⇄ **1/12-oct expert** (D-06). Aggregation strictly by band index (every 4th grid point = 1/3-oct), never nominal Hz.
- **Weighting toggle** (segmented): **dB(A) (default)** ⇄ **dB(C)** — instant, no recompute (D-09). Label sourced from result metadata; if toggle ≠ metadata weighting, show the mismatch note.
- **Coherent/incoherent:** totals **always** show the split; a per-band overlay of the two series is one click away (D-08).
- **Receiver selection:** map click AND a synced receiver list (D-07); selected receiver highlighted with the accent ring in both.
- States: loading (`.loading` 0.55 opacity), empty ("Select a receiver"), stale (badge).

### Color-Scale Editor (`ColorScaleEditor.tsx`, WEB-06/D-04)
- Preset picker (END default + viridis + turbo) + editable break rows (NoizCalc §4.6.5 controls: smallest interval, interval magnitude, number of intervals, ascending, keep-color-sequence).
- Live legend = the single `breaks[]`/`colors[]` source of truth; editing re-contours the cached grid (no re-solve). Weighting label from metadata.
- Validation inline (`.form-error`): monotonic, finite, ≥2 breaks.

### Conditioning controls (`conditioning` store + panel, WEB-05/SVC-06)
- Per-source **Gain** (dB, numeric input), **Delay** (ms, numeric), **Filter** = the **reused Phase-7 `SpectrumEditor`** verbatim (D-11) at 1/1·1/3·1/12 entry.
- **Live debounced recalc ~150 ms** (D-10) drives `compose_gain` → `readout_coherent` MAC; spectra + map update live. No button.
- **Stale badge** (D-12): a `.chip.warn` "Out of date" appears the moment the re-minted blake3 identity ≠ the cached tensor hash. Conditioning edits never stale.
- **409 refusal** (SVC-06): a MAC against a mismatched hash is refused with the reject copy — never silently served.

### Scenario Manager (`ScenarioPanel.tsx`, WEB-12/METX-03/04)
- List of named scenarios (base + user); **"New scenario"** = clone-then-edit (D-13). Friendly met inputs (T/RH/p, Beaufort wind class + direction, temp gradient, downwind worst-case toggle) with an **advanced raw per-azimuth A/B/C** disclosure (D-14). Each scenario computes its own hash-keyed cached tensor; switching is instant.
- **Compare scenarios** → difference-map picker (A vs B) → diverging map (D-16).
- Delete = destructive confirm dialog.

### Export menu (`Export…`, GRID-05/D-20-22)
- Menu (`.menu` idiom): **GeoTIFF** (raster level grid), **GeoJSON** (isophone polygons), **CSV** (spectra: band index + exact Hz). WASM produces bytes → Blob download; nothing leaves the device. Every file carries the attribution footer.

### Isophone map layer (`isophoneLayer.ts`)
- MapLibre **fill layer** (never a heatmap raster — D-02), below the full-styled scene objects (D-18). Legend docked over the map (bottom-left `.map-readout` idiom), showing break values + weighting.

### Info-button (see next section) — on **every** interactive control across all panels.

## Info-Button Help Contract (D-23/D-24/D-25)

- **Component:** `web/src/help/InfoButton.tsx` — a reusable `<InfoButton controlId="…"/>`: a 16px "i" glyph (new `info` icon in `icons.ts`, `--color-text-dim` resting → `--color-text-muted` hover, accent on focus), placed **immediately after the control's label** (right-aligned in the label cell for `.field-row`; trailing the panel-header title for panel-level help). Touch target ≥ the glyph via padding; `:focus-visible` accent ring.
- **Presentation (discretion, decided):** a **popover** for short controls anchored to the button, with a **"More" affordance that opens the docked right-rail help panel** for the long multi-paragraph body — popover for glance, side-panel for depth. One component, two depths.
- **Placement contract:** EVERY interactive control (input, toggle, segmented control, slider, button, preset picker, break row, receiver row) renders exactly one `<InfoButton>`. New Phase-11 controls **and** a retrofit of all Phase-7/8/9/10 panels (scene editor / palette / inspector, import, weather, calc).
- **Help catalog shape** (`web/src/help/catalog.ts`): a typed `Record<ControlId, HelpEntry>` where `HelpEntry = { title: string; body: string[]; citations: Citation[] }` — structured data, NOT text in JSX (D-25). `ControlId` is a string-literal union; a control cannot render an `<InfoButton>` for an id not in the union.
- **Content rules (D-24):** multi-paragraph — what the control does, every option/range/unit/default, and the Nord2000 / NoizCalc rationale. Cite **AV 1106/07 by report number** and **TI 386** in `citations`. **⚠ NEVER paste AV 1106/07 text** (copyrighted — explain in our own words). English-only.
- **Coverage check** (`web/src/help/coverage.test.ts`, D-25): enumerate all `ControlId`s and assert each has a catalog entry; a control without help **FAILS** the test. Derive the id set from the `ControlId` type so a new control can't ship without a help entry.
- **Scope note for the planner (D-25):** isolate this sweep into its own plan/wave — it touches every prior-phase panel and must not serialize the feature work.
