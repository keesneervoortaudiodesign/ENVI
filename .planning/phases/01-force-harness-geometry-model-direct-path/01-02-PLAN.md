<objective>
Give the engine its canonical vocabulary: the semantic 2.5D scene model (GEO-01), the FORCE-case-to-Scene conversion with the lane and hSv/hRv conventions that are the phase's biggest off-by-metres traps (GEO-02), and pure path geometry — azimuth and image-source reflection — validated against hand-computed values through the harness (GEO-03).

Purpose: Every propagation module from plan 01-03 through Phase 4 consumes these types. Locking the conventions now (projected metric CRS, Z-up; source height above FIRST profile point, receiver above LAST; source lane line at x=2.5 m) prevents silent geometric errors that would masquerade as physics bugs in later phases. After this plan the developer can run `cargo test` and watch geometry cases pass green — the first capability flips on.

Output: `envi-engine::scene`, `envi-engine::geometry`, `envi-harness::scene_build`, two hand-computed geometry fixture cases, Geometry capability implemented in the harness.
</objective>
