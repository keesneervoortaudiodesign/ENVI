<objective>
Create the pure-Rust `envi-compute` core crate — the WASM-safe, natively-testable heart of Phase 10 — and factor the tensor-identity types out of `envi-store` so the browser can compute the tensor hash without `std::fs`. This crate holds four physics-free responsibilities: the factored **identity** (manifest + blake3 hash), the **cost** model + guardrail (SC1), the hierarchical **tier** partition (D-05/D-06), and the **SolveJob assembly** that wires the directional-phase seam (SRC-03 Folded Todo).

Purpose: Everything downstream (the wasm cdylib, the worker, the UI) calls into this crate. Keeping it pure Rust means the exact FORCE-validated `solve()` path runs unchanged in the browser, and every non-boundary responsibility is unit-testable with plain `cargo test`. The engine stays byte-identical (D-02).

Output: `crates/envi-compute/` (5 source files + Cargo.toml) and a source-compatible refactor of `envi-store` that re-exports the moved identity types.
</objective>

  <action>
    `cost.rs`: a pure `estimate(area_m2, spacing_fine_m, discrete_points, n_sub) -> CostEstimate` returning `{ receiver_count, tensor_bytes, working_set_bytes, time_estimate }`. Receiver count `N ≈ discrete_points + area_m2 / spacing_fine²` (coarse points are a SUBSET of fine — tiers add NO receivers, D-05). Tensor bytes `= n_sub · N · N_BANDS · BYTES_PER_CELL_PAIR` (use the engine constant; do not hard-code 24). Working set `= n_workers · chunk_receivers · n_sub · N_BANDS · BYTES_PER_CELL_PAIR` (the SC3 bound). Time estimate: a device-adaptive `t_pair`-based extrapolation `(n_sub · N · t_pair) / n_workers` with a conservative built-in `t_pair` default (the calibration probe is Claude's discretion — a fixed default is acceptable here; document it as tunable). Add a `guardrail(estimate, budget) -> Guardrail { level: Ok | Warn | Block, detail }` where `Warn` fires on the soft thresholds (tensor > ~256 MiB per D-06/UI-SPEC, or a long time estimate) and `Block` fires only on a hard budget breach; the warn detail must express "halving the spacing quadruples the cost" (exact since N ∝ 1/spacing²). Keep the acoustic math out — this is byte/receiver arithmetic only (not an SVC-07 violation).

    `tiers.rs`: `partition(fine_spacing_m, lattice_origin, area, discrete_points, coarse_multiples: &[usize]) -> TierPlan`. Because `coarse = k · fine` with integer `k` and a shared lattice origin, a fine lattice point `(i,j)` is also a coarse point iff `i % k == 0 && j % k == 0`. Emit an ordered set of tiers — `[points, coarse (k=10 default → 100 m), fine]` — each tier carrying its receiver id/position list, with coarse a strict subset of fine and fine listing ONLY the ~99% gap points not already in coarse (D-05: no recompute). Keep the multiples list data-driven so an intermediate tier (e.g. 50 m) is a one-line config (RESEARCH Open Q2). Assign global receiver indices in receiver-major order compatible with `solve()`'s non-decreasing-receiver contract.
  </action>
