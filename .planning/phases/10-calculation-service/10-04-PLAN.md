<objective>
Wire the parallel execution core: the caller-side rayon pool driver (`pool.rs`) that shards each tier's receiver axis into disjoint chunk ranges and runs the unchanged engine `solve()` on each — the GRID-02 parallelism — plus the dedicated Web Worker that owns the client-side job lifecycle (initThreadPool, the tier loop, cooperative abort, JobStatus/TierComplete postMessage) and the main-thread client + zustand slice that the UI (10-05) binds to. This is SVC-02's client-side compute-job model.

Purpose: The engine has no internal rayon; GRID-02's "parallelized (rayon)" is satisfied here by a wasm-bindgen-rayon SharedArrayBuffer pool sized to navigator.hardwareConcurrency (D-03), sharding into disjoint files (no shared-mutable sink, no locking). The worker reuses the Phase-6 `JobStatus` vocabulary over `postMessage` (not SSE — the solve is client-side, D-10) and aborts cooperatively at chunk boundaries via a SharedArrayBuffer-visible atomic flag (no `worker.terminate()`, D-11). The CalcPanel UI + Playwright UAT land in 10-05.

Output: `pool.rs` (rayon sharding + cancel + solve_chunk_range) and the `web/src/compute/{opfs,worker,client,cost}.ts` + `web/src/store/calc.ts` integration.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main thread -> worker | Job spec (grid/scene) crosses via postMessage; worker validates and owns compute |
| worker rayon tasks -> OPFS | Disjoint chunk files written under a hex-hash-keyed directory |
| worker -> wasm shared memory | The SharedArrayBuffer cancel flag is visible to all pool threads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04-01 | Denial of service | runaway grid / unbounded receiver set | mitigate | RSS bounded by workers x chunk (SC3); the guardrail (10-01) + OPFS quota check block before submit; cooperative abort frees a stuck run |
| T-10-04-02 | Tampering | OPFS path injection via names | mitigate | Chunk/dir names are hex manifest hash + integer chunk index only; the opfs.ts safeSeg hex-only guard rejects any other component (V12) |
| T-10-04-03 | Tampering | SharedArrayBuffer/atomics misuse | mitigate | A single static AtomicBool cancel flag; no other cross-thread mutable state (disjoint files); cross-origin isolation asserted before initThreadPool |
| T-10-04-04 | Spoofing | worker message origin | mitigate | The worker only accepts a typed submit/cancel message schema and validates crossOriginIsolated; unknown messages are ignored (no arbitrary code path) |
| T-10-04-05 | Denial of service | unbounded job growth in the worker | mitigate | Single client-side job at a time; a new submit supersedes/cancels the in-flight run before starting (mirrors the Phase-11 what-if supersede) |
| T-10-04-06 | Repudiation | silent failure / false green | mitigate | Honest capability-failure state (not a generic failed); cancelled keeps completed tiers; failed carries the real reason (no false Done) |
</threat_model>
