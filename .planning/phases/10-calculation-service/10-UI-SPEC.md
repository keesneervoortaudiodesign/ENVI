# Phase 10 ‚Äî UI Design Contract

> Visual and interaction contract for the Phase-10 **Calculate / Run** surface. Generated by gsd-ui-researcher.
> Scope: submit a calculation, show a pre-run cost estimate + guardrail, watch progressive-tier live
> progress, abort cleanly, and surface a cross-origin-isolation capability failure honestly.
> **OUT OF SCOPE (Phase 11):** results spectra panels, isophone maps, color scales, difference maps.
>
> This phase invents **no** new design language. Every token, class, and layout below already exists in
> `web/src/theme.css` + `web/src/app.css` and is reused verbatim. The new **CalcPanel** is a sibling of
> `ImportPanel` / `WeatherPanel` in the existing right rail.

## Spacing Scale

Reuse the **established metrao3 token scale verbatim** (`theme.css`). It is a 4px-based scale that also
carries a 2px hairline (`--space-0`) and 12px/20px steps ‚Äî this is the shipped system across Phases 7‚Äì9,
so it is adopted as-is. **Do not introduce a parallel 8-point scale.** No new spacing token in Phase 10.

| Token | Value | Phase-10 usage |
|-------|-------|----------------|
| `--space-0` | 2px | segmented-control (`.switch`) inner gap/padding only |
| `--space-1` | 4px | chip‚Üîlabel gaps, list-row vertical padding, readout line gaps |
| `--space-2` | 8px | `.btn-row` gap, control-row gaps, panel inner control spacing |
| `--space-3` | 12px | `.panel-header` horizontal padding |
| `--space-4` | 16px | panel-body padding, right-rail inter-panel `gap` |
| `--space-5` | 20px | (inherited; not required by Phase 10) |
| `--space-6`‚Ä¶`--space-12` | 24‚Äì48px | (inherited; not required by Phase 10) |

Layout heights come from `--row-h` (28px dense rows) and `--row-h-lg` (36px palette/list rows); the 44px
`.btn` min-height (touch target) is retained only on the **Run** and **Abort** primary/destructive actions,
matching the `App.tsx` D-12 rule ("44px retained ONLY on primary Save and destructive actions").

Exceptions: none ‚Äî all values are `var(--space-*)` / `var(--row-h*)`.

## Typography

Reuse `--text-*` tokens (`theme.css`). Exactly **two weights** ship in the system: regular **400** (body,
labels, chips) and medium **500** (`.section-title` only). No new size or weight in Phase 10.

| Role | Token | Size | Weight | Line Height |
|------|-------|------|--------|-------------|
| Panel header (`.panel-header`) | `--text-xs` | 12px | 400 | 1.4 ‚Äî uppercase, `letter-spacing: 0.06em`, `--color-text-muted` |
| Body / control label (`.issue-text`, `.field-label`) | `--text-xs` | 12px | 400 | 1.4 |
| Control text / button (`.btn`, `.input`) | `--text-sm` | 13px | 400 | 1.4 |
| Numeric readout (`.mono`) ‚Äî cost estimate, %, counts | `--text-sm` / `--text-xs` | 13 / 12px | 400 | 1.4, `font-variant-numeric: tabular-nums` |
| Chip (`.chip`) ‚Äî status/tier badges | `--text-xxs` | 11px | 400 | 1.4 ‚Äî uppercase, mono, `letter-spacing: 0.04em` |
| Section title (optional sub-head) | `--text-base` | 15px | 500 | 1.4 |

**Rule:** every computed number (receiver count, tensor MB, time estimate, percent, tier counts) renders in
`.mono` with `tabular-nums` so digits do not jitter while progress updates.

## Color

Reuse metrao3 tokens (`theme.css`). Dark-only. 60 / 30 / 10 split below.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg` `#0b0d10`, `--color-surface` `#14171c` | app background + `.panel` surface (the CalcPanel body) |
| Secondary (30%) | `--color-surface-2` `#1b1f26`, `--color-surface-3` `#232831` | `.input`/`.btn` rest surface, elevated/active rows, active `.seg` |
| Accent (10%) | `--color-primary` `#4ea8ff` | see reserved list below |
| Destructive | `--color-crit` `#ef4444` (`--color-crit-bg` for fills) | **Abort** button (`.btn.danger`), `Failed`/blocked/capability-failure states |

**Accent (`--color-primary`) reserved for ‚Äî and ONLY for:**
- focus rings (`:focus-visible { outline: 2px solid var(--color-primary) }`),
- the **active** option of the spacing/tier segmented control (`.seg.active` text),
- the in-progress emphasis on the running tier (running-tier `.mono` percent + optional bar fill),
- links.

Accent is **not** used as a fill on the Run button (siblings `ImportPanel`/`WeatherPanel` use plain `.btn`
for their run actions ‚Äî CalcPanel matches them; see Open Questions Q1).

**Severity ladder (reused verbatim from `theme.css`, drives every job/tier badge):**

| State | Class | Token | Meaning in Phase 10 |
|-------|-------|-------|---------------------|
| idle / queued | `.chip` (neutral) | `--color-text-muted` | tier not started / job `queued` |
| running | `.chip.warn` | `--color-warn` `#f59e0b` | tier or job `running` (matches ImportPanel `statusSeverity`) |
| done | `.chip.ok` | `--color-ok` `#22c55e` | tier-complete / job `done` |
| failed / blocked | `.chip.crit` + `.dot.crit` (pulse) | `--color-crit` `#ef4444` | job `failed(reason)`, guardrail block, capability failure |
| cancelled | `.chip.off` | `--color-text-muted` on `--color-off-bg` | job `cancelled` |

Crit text sits on `--color-surface`/`--color-bg`, never on `--color-surface-2` (the a11y usage rule already
noted in `app.css` `.reject-banner` / `.form-error`).

## Copywriting Contract

All strings are **English** (hard project rule) and reach the DOM as **React text children** ‚Äî never
`innerHTML`/`dangerouslySetInnerHTML` (the T-08/T-09 innerHTML threat that every sibling panel honors).
`{‚Ä¶}` are runtime values.

| Element | Copy |
|---------|------|
| Panel title (`.panel-header`) | `Calculate` |
| Primary CTA (Run) | `Run calculation` |
| Abort control | `Abort` |
| Spacing field label | `Fine grid spacing` ‚Äî unit chip `m` (`.field-unit`) |
| Derived-tier readout | `Tiers: receiver points ‚Üí coarse {coarse} m ‚Üí fine {fine} m` |
| Cost estimate readout | `{receiverCount} receivers ¬∑ {tensorMiB} MiB tensor ¬∑ ~{timeEstimate}` |
| Guardrail ‚Äî warn | `Fine spacing {fine} m ‚Üí {receiverCount} receivers (~{tensorMiB} MiB). Halving the spacing quadruples the cost.` |
| Guardrail ‚Äî block (crit) | `{receiverCount} receivers exceeds the {budget} budget ‚Äî increase the spacing to run.` |
| Empty state (no project) | `Open a project to run a calculation.` |
| Disabled reason ‚Äî no calc area | `Draw a calculation area to run.` |
| Disabled reason ‚Äî no source | `Add at least one sound source to run.` |
| Tier row labels | `Receiver points` ¬∑ `Coarse grid ({coarse} m)` ¬∑ `Fine grid ({fine} m)` |
| Running progress readout | `{pct}% ¬∑ {message}` (mono, mirrors ImportPanel `{pct}% {message}`) |
| Tier-complete indicator | chip `.chip.ok` reading `done` + `.mono` `{tierReceiverCount} receivers` |
| Terminal ‚Äî cancelled | `Cancelled ‚Äî completed tiers kept.` |
| Terminal ‚Äî failed | `Calculation failed ‚Äî {reason}` |
| Capability failure (crit) | `This browser session is not cross-origin isolated, so the multi-threaded calculation cannot start. Reload the app from its server (it sends the required COOP/COEP headers). If this keeps happening, your browser may not support SharedArrayBuffer.` |
| Abort confirmation | none required ‚Äî abort is **cooperative and non-destructive** (D-11: lands `Cancelled`, keeps already-emitted tiers). A single-click `Abort` is correct; do **not** add a confirm dialog. |

No destructive-with-data-loss action exists in this phase (abort preserves tiers; re-run is idempotent by
manifest hash, D-09), so no typed-name confirm modal (unlike `DeleteProjectDialog`).

## Component Specification ‚Äî `CalcPanel`

New file: `web/src/panels/CalcPanel.tsx`. Mounted in `App.tsx` `aside.right-rail`, as a sibling **after
`WeatherPanel`, before `ValidationPanel`** (workflow order: import ‚Üí weather ‚Üí **calculate** ‚Üí validate).
Follows the exact `<section className="panel" data-testid="calc-panel">` + `<div className="panel-header">`
skeleton of every sibling. Driven by a Zustand store slice (`web/src/store/calc.ts`, sibling of
`store/import.ts` / `store/weather.ts`); **no props**. Every actionable control carries a `data-testid`
(Playwright offline-UAT drives these, as with `import-*` / `weather-*`).

### 1. Setup block (pre-run)

| Element | Class(es) | Behavior |
|---------|-----------|----------|
| Empty state (no open project) | `.empty-state` | shown when `projectId` is null; text = "Open a project to run a calculation." `data-testid="calc-no-project"` |
| Fine-spacing input | `.field-row` > `.field-label` + `input.field-input.input.dense` `type="number"` `min` `step` + `.field-unit` `m` | numeric, **default 10** (D-06). `data-testid="calc-spacing"`. On change ‚Üí recompute derived tiers + cost estimate live |
| Derived-tier readout | `.mono` line | auto-derived coarser multiples (D-06): fine ‚Üí coarse (√ó10 ‚Üí 100 m default), points. Read-only. `data-testid="calc-tiers"` |
| Cost-estimate readout | `.mono` line(s) | receiver count, tensor MiB, rough time ‚Äî recomputed on every spacing/scene change. `data-testid="calc-estimate"` |
| Guardrail banner | `.chip.warn` (soft) / `.chip.crit` (block) | present only when triggered; mirrors `ImportPanel` guardrail (`chip ${blocked ? 'crit' : 'warn'}`). `data-testid="calc-guardrail"` |

**Cost model (Claude's discretion, D ‚Äî record actuals in the estimate readout, not hard-coded here):**
tensor bytes = `subSourceCount √ó receiverCount √ó 105 bands √ó 16 bytes` (Complex<f64>, the frozen tensor
layout). `receiverCount ‚âà calcAreaSquareMetres / spacing¬≤`. The guardrail's "halving spacing quadruples
cost" is exact because receiver count scales with `1/spacing¬≤`. Warn threshold and hard-block budget are
Claude's discretion (see Open Questions Q3) ‚Äî the block state must exist and must disable Run.

### 2. Run control + enable/disable

`<button type="button" className="btn" data-testid="calc-run" disabled={!canRun}>Run calculation</button>`
inside a `.btn-row` (matches `import-run` / `weather-import`). 44px min-height retained.

`canRun` (all must hold): `projectId` present **AND** a `calc_area` feature exists **AND** ‚â•1 `source`
feature exists **AND** guardrail is not `blocked` **AND** `crossOriginIsolated === true` (capability)
**AND** job state is not `running`. When disabled, render the most relevant disabled-reason string
(`.issue-text`, muted) so the block is never silent.

### 3. Live progress (during `running`)

Reuse the `JobStatus` **frozen wire vocabulary** (`wire.ts` L651): `queued` | `running{progress,message}`
| `done` | `failed{reason}` | `cancelled`. Progress arrives via Web-Worker `postMessage` (D-10), not SSE.

- **Overall job status chip:** `<span className="chip ${severity}" data-testid="calc-status">{state}</span>`
  with `statusSeverity` identical in spirit to `ImportPanel`/`WeatherPanel` (`running`‚Üí`warn`, `done`‚Üí`ok`,
  `failed`/`cancelled`‚Üí`crit`/`off`). A `.dot.crit` pulse prefixes the `failed` chip.
- **Overall progress readout:** `.mono` `{pct}% ¬∑ {message}` (`data-testid="calc-progress"`), the exact
  `{pct}% {message}` idiom from `ImportPanel` `LayerRow`.
- **Per-tier rows:** a `.issue-list` (`data-testid="calc-tiers-list"`) of three rows ‚Äî Receiver points,
  Coarse grid, Fine grid ‚Äî each an `li.issue-row`-style row carrying: the tier label (`.issue-text`), a
  status chip (`queued`/`running`‚Üíwarn/`done`‚Üíok), and, when done, a `.mono` `{n} receivers` count. A
  tier that has emitted its tier-complete event (D-07) shows `.chip.ok` `done`. `data-testid="calc-tier-{points|coarse|fine}"`.
- Progressive semantics: coarse points are a strict subset kept as-is (D-05) ‚Äî rows fill top-to-bottom;
  an earlier tier never reverts from `done`.

### 4. Abort control + terminal states

- **Abort button:** `<button type="button" className="btn danger" data-testid="calc-abort">Abort</button>`,
  rendered only while `queued`/`running`. Single click sets the SharedArrayBuffer atomic cancel flag
  (D-11); no confirm dialog. Cooperative ‚Äî lands at a chunk boundary.
- **Cancelled terminal:** status chip `.chip.off` `cancelled` + line "Cancelled ‚Äî completed tiers kept."
  (`data-testid="calc-cancelled"`). Completed tier rows retain their `done` chips.
- **Failed terminal:** status chip `.chip.crit` + `.dot.crit` + `.form-error`/`.issue-text` line
  "Calculation failed ‚Äî {reason}" rendering the real `reason` string (`data-testid="calc-failed"`).
- **Done terminal:** status chip `.chip.ok` `done`; Run re-enabled (idempotent re-run reuses the
  manifest-hashed tensor, D-09). No results UI here (Phase 11).

### 5. Capability-failure surface (cross-origin isolation)

Checked on mount and before Run. When `!self.crossOriginIsolated` or `SharedArrayBuffer` is undefined,
render a persistent crit banner at the top of the panel body (`.chip.crit` block or `.form-error` line,
`data-testid="calc-capability-error"`) with the capability-failure copy, and force `canRun = false`.
**No silent failure** ‚Äî compute genuinely cannot start without the SAB thread pool (D-03/D-04). This is a
distinct, honest state, not a generic `failed(reason)`.

## States (State Matrix)

| # | State | Trigger | Visual |
|---|-------|---------|--------|
| S0 | No project | `projectId == null` | `.empty-state`; all controls absent |
| S1 | Not cross-origin isolated | `!crossOriginIsolated` | `calc-capability-error` crit banner; Run disabled |
| S2 | Incomplete scene | no calc area / no source | Run disabled + disabled-reason line |
| S3 | Ready | project + area + source + isolated + not blocked | cost estimate shown; Run enabled |
| S4 | Guardrail warn | spacing cost high but under budget | `.chip.warn` guardrail; Run **still enabled** |
| S5 | Guardrail block | cost over budget | `.chip.crit` guardrail; Run **disabled** |
| S6 | Queued | Run clicked, pool spinning up | status chip `queued`; Abort visible |
| S7 | Running (per-tier) | worker emits progress/tier-complete | warn status chip + `{pct}%` + filling tier rows; Abort visible |
| S8 | Done | all tiers emitted | ok status chip; tier rows all `done`; Run re-enabled |
| S9 | Cancelled | Abort ‚Üí chunk-boundary land | off chip + "completed tiers kept"; done tiers retained |
| S10 | Failed | worker error | crit chip + pulse + reason line |

## Interaction Rules

1. **Live recompute:** editing `Fine grid spacing`, the calc area, or the source set recomputes the derived
   tiers + cost estimate + guardrail synchronously (no spinner) ‚Äî mirrors `WeatherPanel`/`ValidationPanel`
   deriving from store state. Debounce numeric typing if needed, but never block the input.
2. **Run is idempotent by identity:** a re-run of an unchanged scene reuses the manifest-hashed OPFS tensor
   (D-09); the UI need not warn or confirm.
3. **Abort is one click, non-destructive, no dialog** (D-11). Never call `worker.terminate()`.
4. **Tiers only advance:** a tier chip goes `queued ‚Üí running ‚Üí done` and never regresses; a cancel freezes
   the ladder at the last completed tier.
5. **Every number is `.mono` + `tabular-nums`** so progress/counts do not reflow.
6. **All dynamic strings are React text children** ‚Äî reason, guardrail detail, counts (innerHTML forbidden).
7. **Focus + keyboard:** Run/Abort are real `<button>`s reachable by Tab, actuated by Enter/Space, with the
   `:focus-visible` primary outline already defined in `theme.css`.

## Accessibility

- Reuse the theme `:focus-visible { outline: 2px solid var(--color-primary); outline-offset }` on Run,
  Abort, spacing input, and segmented options (already the system default; do not override).
- 44px min touch target retained on Run + Abort (`.btn`).
- Status is conveyed by **label text, not color alone**: every chip carries its literal state word
  (`running`/`done`/`failed`/`cancelled`), matching the `save-conn` "state-coloured TEXT label" precedent.
- The live progress readout region should be an ARIA live region (`aria-live="polite"`) so screen readers
  announce tier-complete + percent without stealing focus; the `failed`/capability crit banner may use
  `role="alert"`. `.dot` glyphs are `aria-hidden="true"` (they duplicate the text state).
- Color pairs use the established crit-on-surface (never crit-on-surface-2) contrast rule.

## Open Questions (non-blocking ‚Äî recommended defaults marked ‚òÖ)

- **Q1 ‚Äî Run button emphasis.** ‚òÖ Plain `.btn` (matches the sibling run actions `import-run` /
  `weather-import`, keeps accent reserved for state). Alternative: `.btn.primary` accent fill because Run is
  the workflow's headline action ‚Äî the token exists. Chose plain `.btn` for cross-panel consistency; flip
  only if the product owner wants Run visually elevated above Import/Weather.
- **Q2 ‚Äî Progress bar vs. text percent.** ‚òÖ Text-percent readout (`{pct}% ¬∑ {message}`), the shipped
  `ImportPanel` idiom ‚Äî the theme has **no** progress-bar primitive, and inventing one would break the
  reuse mandate. Optional enhancement: a thin token-only bar (a `div` at `width:{pct}%` in
  `--color-primary` over `--color-surface-2`, 3px, `--radius-full`) ‚Äî additive, still token-only; defer
  unless requested.
- **Q3 ‚Äî Guardrail thresholds + time model.** Explicitly Claude's discretion (CONTEXT ¬ßClaude's Discretion).
  ‚òÖ Recommend: warn when the estimated tensor exceeds ~256 MiB, hard-block above the OPFS/working-set budget
  the executor derives from `CountingSink` + `navigator.hardwareConcurrency`; time estimate from a
  measured per-receiver-per-band cost calibrated on the first tier. Finalize during planning against the
  real memory math ‚Äî the states (warn/block) are contractually required regardless of the numbers.
- **Q4 ‚Äî Intermediate preview tier.** D-06 allows an optional intermediate tier between points and fine.
  ‚òÖ Ship the two-tier default (coarse 100 m ‚Üí fine 10 m); if an intermediate (e.g. 30 m) is added, it is
  simply one more `.issue-row` in the tiers list ‚Äî no contract change.
