<objective>
Close the "SCN/WEB objects are silently inert" gap on the store side (D-01): add typed, validated `IsolationSpectrumDto` and `ForestParamsDto` to `envi-store`, plus the single shared band-index interpolation core (D-05/D-06) that the read-path `r_db[105]` derivation, the Phase-7 interpolation endpoint, and `PUT /scene` validation all consume so they cannot diverge.   <action>
    Create module `interpolate` (declare `pub mod interpolate;` in lib.rs). Define `pub enum Resolution { Octave, Third, Twelfth }` and `pub fn interpolate(resolution: Resolution, values: &[f64]) -> Result<[f64; N_BANDS], StoreError>`. Anchor grids per D-05/07-RESEARCH Pattern 6: Octave → 9 anchors at grid indices `4 + 12*k` (k in 0..9, i.e. 4,16,…,100); Third → 27 anchors at `4*k` (0,4,…,104); Twelfth → identity over all 105. Reject any `values.len()` not matching the resolution's anchor count with `StoreError::BadBandCount { got }`; reject any non-finite value with `StoreError::NonFinite { what }`. Interpolate LINEARLY IN BAND INDEX between bracketing anchors (equal to linear in log-frequency, SCN-03). Extrapolation outside the spanned range (octave leaves bands 0..3 and 101..104 unspanned) uses FLAT-HOLD clamp to the nearest endpoint anchor value (adopt A1 explicitly; document it in the module header). Clamp every output to `[0.0, 1000.0]` (the `IsolationSpectrum` domain, `MAX_R_DB`). Add a `//!` Module I/O header documenting inputs (resolution + values), output (dense [105]), the exact stride math, and the flat-hold decision. Compare BY BAND INDEX only — never nominal Hz (Pitfall 4).
  </action>
  <verify>
    <automated>cargo test -p envi-store interpolate -- --nocapture</automated>
  </verify>
  <acceptance_criteria>
