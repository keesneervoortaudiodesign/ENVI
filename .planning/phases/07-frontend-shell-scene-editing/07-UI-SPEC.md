# Phase 7 — UI Design Contract

> Visual and interaction contract for the ENVI scene-authoring frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **The look is already decided (D-11).** This document does NOT choose a visual identity — it specifies precisely how ENVI's screens, the 9-kind object palette, the isolation-spectrum editor, and every state are built *from* the adopted `metrao3` token set. Every value below cites an existing `--token`. Where a needed value has no token, it is reported as a **TOKEN GAP**, never invented.

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** — hand-rolled CSS on adopted tokens (D-11). No shadcn, no MUI/Chakra/Tailwind (constraint). |
| Preset | not applicable |
| Component library | **none** — the `metrao3` primitives (`.panel`, `.panel-header`, `.btn`, `.input`, `.chip`, `.dot`, `.section-title`, `.mono`) hand-composed in React/TSX |
| Token source | `web/src/theme.css` ← copied **VERBATIM** from `metrao3/crates/metrao3-web/ui/src/theme.css` (embedded, system-font, dark-only variant). READ-ONLY ancestor; never edited in-place. |
| Component CSS | `web/src/app.css` — ENVI-specific layout layered on tokens; **no new color/spacing/radius tokens** |
| Icon library | **none** — inline SVG built via DOM APIs, the `icons.ts` pattern (0..16 viewBox, `stroke="currentColor"`, `stroke-width="1.4"`, round caps/joins, `aria-hidden`). No icon font, no CDN, never `innerHTML`. |

| Theme mode | Dark-only. `--color-bg #0b0d10` is the base; the MapLibre basemap is `OpenFreeMap styles/dark` so the chrome/map luminance is continuous (D-13/D-13a). |

## Spacing Scale

The adopted `metrao3` scale IS the contract (D-11). Declared tokens (do not invent intermediate values):

| Token | Value | Usage in ENVI |
|-------|-------|---------------|

| `--space-1` | 4px | Icon↔label gap, chip cluster gap, palette dot spacing |
| `--space-2` | 8px | Compact inspector row gaps, button-row gap, topbar item gap fallback |
| `--space-3` | 12px | Panel-header padding (vertical), key/value cell padding |
| `--space-4` | 16px | Default panel padding, topbar horizontal padding, rail↔map gap |
| `--space-5` | 20px | Reserved (matches token; used for spectrum-editor internal padding) |
| `--space-6` | 24px | Section breaks inside the spectrum editor overlay |

**Exception note:** `--space-0` (2px) is below the 8-point default the generic template assumes. It is an *adopted* token (segmented-control gaps), not an ENVI invention, and is used only for hairline internal gaps. All ENVI-authored spacing otherwise uses `--space-1`..`--space-8`.

## Typography

The adopted type scale + the theme's **two** weights (400 default, 500 medium — the theme defines no 600/700). Body default is `--text-sm` (13px) per `html`.

| Role | Token / Size | Weight | Line Height | Usage |
|------|--------------|--------|-------------|-------|
| Body | `--text-sm` 13px | 400 | 1.4 (theme default) | Inspector values, list rows, general text |
| Label / caption | `--text-xs` 12px | 400 | 1.4 | Inspector field labels, `.panel-header`, key cells |
| Micro-label | `--text-xxs` 11px | 400 (mono in chips) | 1.4 | `.chip` text, bar-value micro-labels, spectrum axis tick labels |
| Section title | `--text-base` 15px | 500 | 1.4 | `.section-title` — panel/overlay titles |
| Numeric readout | `--text-sm` 13px `.mono` | 400/500 | 1.4 | 105-band spectrum table, coordinate/height/SPL values (`font-variant-numeric: tabular-nums`) |
| Emphasis value | `--text-xl` 20px `.mono` | 500 | **1.15** (see note) | Large single values (e.g. selected-object SPL headline, dB anchor readout) |

**Weights:** exactly two — 400 (regular) and 500 (medium). No bold/600 exists in the token set; do not introduce one.

**Raw-value note (the one intentional exception):** `line-height: 1.15` on the emphasis-value row is the **sole raw typographic value** in this document that cites no `--token` — the token set defines no line-height token, and this value is copied verbatim from the adopted `metrao3` `.hero .tile .value` / `.vtile .value` rule (`styles.css`), where a tight 1.15 keeps a large tabular-nums readout visually compact. It is an *adopted* value, not an ENVI invention. Every other typographic value uses a token or the theme default 1.4. This is the only line-height that departs from 1.4; the "nothing invented" claim below holds because this value is inherited from the token source, not authored here.

## Color

The theme is the **entire** palette. There are exactly five chromatic hues, two of which are reserved for validation state, plus a text ramp and a surface ramp. The 60/30/10 split maps onto the dark ops-console surfaces:

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (~60%) | `--color-bg` / basemap | `#0b0d10` | App background + dark map canvas (continuous luminance) |
| Secondary (~30%) | `--color-surface` → `--color-surface-3` | `#14171c`→`#232831` | Panels, rails, project bar, inputs, rows (layered) |
| Accent (~10%) | `--color-primary` | `#4ea8ff` | See reserved-for list below |
| Destructive | `--color-crit` | `#ef4444` | Delete project, hard-reject message, `crit` validation |

**Accent (`--color-primary` #4ea8ff) is reserved for, and ONLY for:**
1. The active tool in the object palette (selected-tool highlight).
2. `:focus-visible` outlines (2px, `outline-offset: 2px` — the theme rule).
3. Primary action buttons (`.btn.primary` — Save; `#06121e` text on primary fill).
4. The current selection halo on the map + the selected inspector row.
5. The `source` kind and the `calc_area`/`elevation_*` map hue (these are *kind identities*, disambiguated by geometry+symbol — see the palette rationale).
6. The live-preview curve line in the spectrum editor.

**Severity vocabulary (the validation language, D-11) — NOT resting kind colours:**

| Token | Value | Meaning |
|-------|-------|---------|
| `--color-ok` | `#22c55e` | Valid / measurement (green) |
| `--color-warn` | `#f59e0b` | Non-geometric issue (semi-transparent screen w/o spectrum; forest zero density) |
| `--color-crit` | `#ef4444` | Rejected crossing ground zone (D-07); hard error |
| `--color-off` | `#5a6270` | Neutral / recessive / inactive |
| `--color-info` | `#4ea8ff` | == primary; informational (elevation metadata) |

**Reserved-state rule (load-bearing):** `--color-warn` (amber) and `--color-crit` (red) are used by **no kind at rest**. They appear on the map only as a *state overlay* on an otherwise-valid object, so "turning amber/red" always reads as a state change, never a kind identity. This is how the palette avoids collision between kind hue and severity meaning despite the tiny hue set.

## Object Palette — All 9 Locked Kinds

`KINDS` (frozen, `geojson.rs:38`): `source, receiver, wall, building, forest, ground_zone, elevation_point, elevation_line, calc_area`.

**Palette design principle.** The token set offers only three *resting* chromatic hues (blue `--color-primary`/`--color-info`, green `--color-ok`, grey-blue `--color-off`) once warn+crit are reserved for state — far fewer than 9. Kinds are therefore separated by the **product of hue × geometry-type × symbol**, and hue is only ever *reused* across different geometry types, never within one. Grouped by Terra Draw mode:

- **Points:** source (blue), receiver (green), elevation_point (white) — three distinct hues.
- **Lines:** wall/screen (white solid), elevation_line (blue dashed) — distinct hue + dash.
- **Polygons:** building (muted-grey), forest (green), ground_zone (off-grey dashed), calc_area (blue outline, no fill) — separated by fill/opacity/stroke.

| Kind | TD mode | Map render (tokens only) | Icon glyph (0..16 viewBox, stroke) | Selected | Hover | Invalid state |
|------|---------|--------------------------|-------------------------------------|----------|-------|---------------|
| **source** | point | Filled disc `--color-primary`, 2 radiating arcs; `--color-text-strong` core dot | central dot + 2 concentric arcs radiating up-right (like `wifi` glyph reused conceptually) | +`--color-primary-hover` halo, stroke-width ↑ | brighten to `--color-primary-hover` | n/a (source has no draw-time reject) |
| **receiver** | point | Ring/target `--color-ok`, cross-hair centre | circle + short crosshair ticks N/E/S/W | `--color-primary-hover` halo | brighten | n/a |
| **wall / screen** | linestring | Solid line, `--color-text` `#e4e6eb`, 2px; **semi-transparent variant:** double-stroke with `--color-info` inner tint | short slash "／" (single stroke line) | primary halo along line | thicken | **`--color-warn`** line when marked semi-transparent **without** a spectrum (WEB-04 warn) |
| **building** | polygon | Fill `--color-text-muted` @ ~0.22 alpha, stroke `--color-border-strong`; carries an eaves-height label (`.mono`) | rectangle outline "▭" with a small roof tick | primary stroke + fill lift | stroke → `--color-border-strong` | edges that are semi-transparent w/o spectrum → per-edge `--color-warn` stroke |
| **forest** | polygon | Fill `--color-ok` @ ~0.18 alpha, stroke `--color-ok` @ 0.5 | tree/clover "♣" (3 arcs + stem) | primary stroke overlay | fill alpha ↑ | **`--color-warn`** stroke when mean density = 0 (WEB-04 warn) |
| **ground_zone** | polygon | Fill `--color-off` @ ~0.16 alpha, **dashed** `--color-off` stroke (a *region*, not a structure) | filled-corner region "▨" (hatch strokes in a box) | primary dashed stroke | fill alpha ↑ | **`--color-crit`** flash on the *existing* zone during a partial-crossing reject (D-07, transient) |
| **elevation_point** | point | Small diamond `--color-info`, `--color-text` Z-label beside it | diamond "◆" outline + tiny "z" tick | primary halo | brighten | n/a |
| **elevation_line** | linestring | **Dashed** line `--color-info`, 1.4px (contour/breakline) | wavy line "∿" | primary dashed halo | brighten | reject if it interior-crosses another breakline (server `spade` guard → `crit` chip; see Validation) |
| **calc_area** | polygon | **No fill**, dashed outline `--color-primary`, 1.4px (the domain boundary) | dashed square "⬚" | solid primary outline | outline brighten | n/a |

**Legibility on the dark basemap (`OpenFreeMap styles/dark`, ~`--color-bg` luminance):** every resting kind hue clears the WCAG **3:1 graphic** threshold against `--color-bg` (see Accessibility). The two closest polygons — building (`--color-text-muted` grey) and ground_zone (`--color-off` grey-blue) — are disambiguated by **opacity + dashed stroke + building's height label**, not by hue alone. White (`--color-text`) is used only for wall lines and the elevation_point diamond outline, where its brightness reads as "structural / measured", not as a severity signal.

**Semi-transparent screen/building treatment (WEB-08/SCN-01, WEB-09/SCN-02).** A wall/edge that *has* an isolation spectrum renders with the double-stroke `--color-info` inner tint (acoustically special). A wall/edge marked semi-transparent but *lacking* a spectrum renders `--color-warn` and raises a `warn` chip. An opaque wall stays solid `--color-text`. This ties the map treatment directly to the validation language.

## Isolation-Spectrum Editor (WEB-10, SCN-03) — **both curve AND table**

**Decision: ship BOTH surfaces, side-by-side, in the overlay.** A 105-band table alone is unusable for shape; a curve alone cannot be precisely typed. The two views are the direct visual analogue of D-06 (authored persisted / `r_db[105]` derived):

- **Curve view (left, fixed height ~200px):** the full derived 105-point grid, plotted as **dB (R, isolation) vs band INDEX 0..104 — never Hz**. X-axis tick labels at the **1/3-octave centre indices** (stride 4: `0,4,8,…,104`), each label showing the nominal Hz as *display-only* text (`--color-text-dim`, `--text-xxs`, from `GET /meta/freq-axis` — never hardcoded). Gridlines `--color-border`; axis labels `--color-text-dim`. Reuses the `.octave` bar-chart idiom from `styles.css` conceptually (fixed node set, mutate height).
  - **Server preview** (`POST /meta/interpolate-spectrum`, D-05) drawn as a continuous line in `--color-primary`.
  - **Authored anchors** drawn as markers on top in `--color-text-strong` (9 / 27 / 105 markers by resolution). The visual gap between the primary preview line and the white anchor markers *is* the "authored vs interpolated" distinction.
- **Table view (right, `.mono` + `.dense` rows `--row-h` 28px, scrolls):** one row per **authored anchor** at the active resolution (9 octave / 27 third / 105 twelfth), `tabular-nums`. Columns: band index · nominal Hz (display-only) · editable dB `.input` (dense, `--row-h`). Editing a cell updates the store and re-requests the preview (debounced).

**Resolution selector.** A three-way segmented control (`.switch` pattern from styles.css) — `1/1-octave (9)` · `1/3-octave (27)` · `1/12-octave (105)`. Switching resolution re-projects the *authored* representation (D-06 lossless): octave↔third↔twelfth changes which anchors the table shows; because only `authored:{resolution,values}` is stored, switching is non-destructive.

**Promote-to-authored@twelfth (explicit, never silent).** When the resolution is octave/third and the user edits an *individual 1/12 band* (clicks a non-anchor point on the curve, or switches to `1/12` and edits any cell), the editor: (1) calls `interpolate-spectrum` to materialize the current 105 grid, (2) sets `authored = { resolution: "twelfth", values: [105] }`, (3) applies the edit, and (4) shows a one-line notice (`--color-info`, `.chip.info`): **"Switched to 1/12-octave — all 105 bands are now individually editable."** The coarse values are promoted, not discarded (D-06).

**Presets (Claude's discretion):** OPTIONAL. If shipped, a small dropdown of typical glazing/wall `R(f)` presets that populate `authored.values` at their native resolution. Not required for SC3.

**States:**

| State | Presentation |
|-------|--------------|
| Empty (no spectrum yet) | `.empty-state`: "No isolation spectrum. Enter values below or pick a resolution to begin." + resolution selector active |
| Loading (awaiting preview) | Curve line dimmed to `--color-text-dim`; table inputs enabled (optimistic); a `.chip.off` "interpolating…" |
| Error (interpolate 4xx/network) | `.form-error` (`--color-crit`, `--text-xs`) with the server `detail` string rendered as **text only** (path-redacted server-side); table keeps last-good values |
| Valid / applied | Preview line `--color-primary`, anchors `--color-text-strong`; no chip |

## Per-Façade Assignment (WEB-09, SCN-02)

Keyed by **stable per-edge UUIDs** (D-02), reconciled in the store's `applyTerraDrawChange` via the Pattern-5 ring diff — never by vertex index.

**Map affordance.** With a building selected, its footprint edges become individually hoverable: hovering an edge highlights that single segment in `--color-primary-hover`; clicking selects it. A selected edge draws a `--color-primary` overlay along that segment. Edges are the click targets — vertices remain the geometry handles.

**Panel affordance (inspector).** The building inspector lists its edges as rows (one per `edge_id`, `--row-h` dense):

| Edge (short UUID `.mono`) | Isolation | Chip |
|---------------------------|-----------|------|
| `c3d4…` (selected → primary row) | **Override** → "Edit spectrum" | `.chip.info` "OVERRIDE" |
| `a1b2…` | Inherits `default_isolation` | `.chip.off` "DEFAULT" |

- **Inherits vs overrides distinction:** an inheriting edge shows a `.chip.off` "DEFAULT" and renders in the base wall treatment; an overriding edge shows a `.chip.info` "OVERRIDE" and renders with the double-stroke `--color-info` tint on the map. Clicking an edge row (or the map segment) opens the spectrum editor for that `edge_id`.
- **`default_isolation` presentation:** a first row above the edge list — "Building default" — editable once, applied to every edge whose `facade_isolation` has no entry. Editing the default updates all inheriting edges live.

## Validation Panel (WEB-04, SC2) — two DISTINCT surfaces

These are different things; the spec keeps them apart deliberately (D-07 resolves the SC2 ambiguity):

### Surface A — persistent validation panel (right rail, `.panel`)
Non-geometric issues on objects that **exist**. Each issue is a clickable row: a `.chip` (crit/warn) + a text description (rendered as **text only**) + the object's short id.

| Issue | Chip | Trigger |
|-------|------|---------|
| Wall/edge semi-transparent, no spectrum | `.chip.warn` | WEB-04 |
| Forest mean density = 0 | `.chip.warn` | WEB-04 |
| Elevation breaklines interior-cross (server `spade` reject) | `.chip.crit` | D-08 guard |

**Behaviour:** click a row → select the offending object in the store + **zoom-to-fit** its geometry on the map + open its inspector. `crit` uses the `.dot.crit` pulse animation (theme). Empty state: `.empty-state` "No issues — the scene is valid."

### Surface B — transient hard-reject message (map-anchored, `--color-crit`)
A partially-crossing `ground_zone` (D-07) is **hard-rejected at draw time**: geometry reverts to last-valid, and a **transient** crit banner appears top-center over the map: *"Ground zone can't partially overlap another. Containment is allowed."* + a **"Zoom to conflicting zone"** action. The rejected polygon has **no object to select** — the zoom targets the *existing* zone it crossed. Auto-dismiss on the next action or after a short timeout. This banner is **never** a row in Surface A.

## Ground-Zone Inspector Fields (WEB-04)

| Field | Control | Notes |
|-------|---------|-------|

## Last-Object Property Inheritance (WEB-04)

Mirrors the NoizCalc rule (TI 386 §3.2: *"Object properties are always taken over from the last object of the same type entered"*). Assumed default: **per-kind, session-scoped** (Claude's discretion).

- On finishing a new object of kind K, its non-geometric properties are seeded from the last-committed object of kind K this session (e.g. a new ground_zone inherits the previous zone's impedance A–H + roughness; a new forest inherits density/radius/height; a new wall inherits its spectrum-or-none).
- **Never a surprise:** the inspector shows a `.chip.info` **"inherited from last {kind}"** on each seeded field until the user edits it (the chip clears on edit). First-of-kind objects get the documented per-kind defaults (e.g. ground `--` = settings `default_ground_class 'D'`) with a `.chip.off` "default".

## Destructive Action — Delete Project

"Delete project" lives in the project-bar `⋯` overflow and keeps the 44px target (D-12) because a misclick is costly. It is **irreversible on disk**: `envi-store` `ProjectStore::delete` removes the whole project folder *and* evicts its calc records — there is no undo and no trash. A plain "Are you sure?" is therefore too weak.

**Affordance: typed-name confirmation dialog** (tokens + primitives only).

- **Trigger:** `⋯` → "Delete project…" opens a centered modal (`.panel` + `--shadow-lg`, on a `rgba(11,13,16,0.92)` scrim reusing the `.refreshing` overlay idiom from `styles.css`).
- **Header:** `.section-title` "Delete project" + a `.chip.crit` "IRREVERSIBLE".
- **Body copy (text nodes only, project name rendered via `textContent`):** *"This permanently deletes “{name}” from disk — its scene, settings, and all calculation records. This cannot be undone. Type the project name to confirm."*
- **Confirm gate:** an `.input` (44px) into which the user must type the exact project name. String compared client-side against the store's canonical name (no server call, no acoustic math). The confirm button stays `disabled` (theme `.btn:disabled`, opacity 0.5) until the typed text matches exactly.
- **Actions (`.btn-row`):** `Cancel` (`.btn`, 44px, default focus) + `Delete project` (`.btn.danger`, 44px, `--color-crit` — enabled only on exact match). `Esc` cancels; focus opens on Cancel, never on the danger button.
- **On confirm:** issue the delete, then route to the empty/no-project state (project bar shows "No project — [New] or [Open]").

**States:**

| State | Presentation |
|-------|--------------|
| Idle (name not yet matched) | Danger button `disabled`; helper line `--color-text-muted` `--text-xs` "Names must match exactly." |
| Ready (exact match) | Danger button enabled (`.btn.danger`) |
| Deleting | Danger button shows `.chip.off` "deleting…", both buttons disabled (no double-submit) |
| Error (delete failed) | `.form-error` (`--color-crit`, `--text-xs`) with the server `detail` string rendered as **text only** (path-redacted server-side); dialog stays open, name field retained so the user can retry |
| Success | Dialog closes; project bar → empty state |

The same typed-name pattern is **not** required for lesser destructive acts (Delete selection / `Delete` key on an object) — those are in-scene, autosave-recoverable via the store, and use the plain `Delete` key + selection revert.

## Accessibility + Safety

### Contrast (computed WCAG 2.1 relative-luminance ratios)

Text threshold 4.5:1 · graphic/UI-component threshold 3:1. Backgrounds: `--color-bg #0b0d10`, `--color-surface #14171c`, `--color-surface-2 #1b1f26`.

| Foreground | vs `--color-bg` | vs `--color-surface` | vs `--color-surface-2` | Verdict |
|------------|-----------------|----------------------|------------------------|---------|
| `--color-text` #e4e6eb | **15.6:1** | 14.4:1 | — | text PASS everywhere |
| `--color-text-muted` #8b94a3 | **6.4:1** | 5.9:1 | 5.4:1 | text PASS (labels) |
| `--color-primary` #4ea8ff | **7.8:1** | 7.2:1 | 6.6:1 | text + graphic PASS |
| `--color-ok` #22c55e | **8.5:1** | 7.9:1 | — | text + graphic PASS |
| `--color-warn` #f59e0b | **9.1:1** | — | — | text + graphic PASS |
| `--color-crit` #ef4444 | **5.2:1** | 4.8:1 | 4.4:1 | text PASS on bg/surface; **⚠ 4.4:1 < 4.5 on `--color-surface-2`** |
| `--color-off` #5a6270 | **3.2:1** | 2.9:1 | — | **graphic-only**: PASS 3:1 on bg; **⚠ 2.9:1 on surface** |
| `--color-text-strong`→primary btn text #06121e on `--color-primary` | **7.5:1** | — | — | button text PASS |

**Two flags (both mitigated, no token invented):**
1. **`--color-crit` as plain text on `--color-surface-2` = 4.4:1** (just under 4.5). Mitigation: crit text only ever appears inside `.chip.crit` (which sits on its own `--color-crit-bg` overlay tint) or on `--color-bg`/`--color-surface` — never as plain body text on `--color-surface-2`. The `.dot.crit`/stroke uses of crit are graphics (3:1 met). Hold this as a usage rule.
2. **`--color-off` = 3.2:1 on bg, 2.9:1 on surface.** Mitigation: `--color-off` is used **only as a graphic** (ground_zone map fill/stroke over the dark *basemap* ≈ bg luminance → 3:1 met; `.dot.off`/`.chip.off`). It **never carries text meaning** — chip text uses `--color-text-muted` (5.4:1). Ground_zone legibility is further aided by its dashed stroke, not fill contrast alone.

### Never-`innerHTML` (hard invariant, D-11)

All server-derived and user-authored strings — **project names, object ids, validation messages, spectrum-editor errors, and error `detail` bodies** — reach the DOM as React text children / `textContent` only. **No `dangerouslySetInnerHTML`, ever.** SVG icons are DOM-constructed (`icons.ts` pattern), never parsed from a string. Error bodies are already path-redacted server-side (Phase 6); the frontend still treats them as untrusted text.

## TOKEN GAPS (reported, not invented)

| # | Gap | Impact | Recommended resolution (existing tokens only) |
|---|-----|--------|-----------------------------------------------|
| G1 | **Palette hue exhaustion.** After reserving `warn`+`crit` for state, only 3 resting chromatic hues exist (blue, green, off-grey). All 9 kinds fit *only* by combining hue × geometry × symbol. | A future **6th polygon kind** (e.g. Phase-8 roads/railways/berms) would have no free resting hue. | Accept for Phase 7 (all 9 kinds accommodated). If a new polygon kind lands later, a **new kind-hue token** will be needed then — flag to the roadmap; do not pre-invent now. |
| G2 | **No dedicated "selection" hue** distinct from `--color-primary`. `source`, `elevation_*`, and `calc_area` already rest on primary/info blue. | A selected blue object can't change *colour* to show selection. | Show selection via **stroke-width increase + a `--color-primary-hover` #79bcff halo** (both existing tokens), not a colour swap. Sufficient; no new token. |
| G3 | **No "dirty/unsaved" semantic token.** Reusing `--color-warn` for "Unsaved" overloads the validation-warn hue. | Minor ambiguity risk. | Mitigated by location (project bar) + text label + no pulse (see State Matrix). Accepted overload, **not** a request for a new token. |
| G4 | **No plot/axis-specific tokens** (gridline, axis label, series colour) for the spectrum curve. | Curve must borrow. | Reuse `--color-border` (gridlines), `--color-text-dim` (axis labels), `--color-primary` (preview line), `--color-text-strong` (authored anchors). No gap in practice. |

*(No hex colour, spacing, radius, shadow, font-size, or row-height value in this document is ENVI-invented — every one cites a `theme.css` token. The **two** raw non-token values — the `1.15` emphasis line-height and the `rgba(11,13,16,0.92)` delete-dialog scrim — are copied **verbatim** from the adopted `metrao3` CSS source (`styles.css` `.vtile .value` and `.refreshing` respectively), i.e. inherited from the token source, not authored here.)*
