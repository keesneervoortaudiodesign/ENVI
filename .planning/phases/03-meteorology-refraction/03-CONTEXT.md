# Phase 3: Meteorology & Refraction - Context

**Gathered:** 2026-07-08

<domain>
## Phase Boundary

Phase 3 delivers the **refraction machinery that makes Nord2000 worth building**: the log-lin sound-speed profile `c(z) = A·ln(z/z₀+1) + B·z + C` collapsed to an **equivalent-linear** profile with guarded numerics (f64, ξ singularity clamps, cancellation-safe Δτ), the **frequency-dependent ground variant** (CalcEqSSPGround, f_L/f_H log-interpolation at the 1/12-octave points), **reflection-path coefficient split** (A₁/B₁ before, A₂/B₂ after), **per-azimuth A derivation** (isotropic temperature part once, projected wind added per bearing), the **weather-input routes** (Route 1 weather classes, Route 2 FORCE surface met → A/B, Route 3 Monin–Obukhov reconstruction + LSQ fit), the **upward-refraction shadow zone**, and the **fluctuating-refraction coherence coefficient F_τ** — all validated on the refraction FORCE cases.

Requirements in scope: **ENG-05, ENG-06, ENG-08, MET-01, MET-02, MET-03, MET-04, MET-05, MET-06** (9 total).

**Not in this phase:** the complex transfer tensor store + directional sources + full FORCE numeric pass (Phase 4); any GIS/weather-API ingestion (v2 METX). Geometry and meteorology inputs come from FORCE test-case files, exactly as Phases 1–2.
</domain>

<decisions>
## Implementation Decisions

### Validation gate strategy
- **D-01:** Refraction gates for numeric green via the **same oracle+anchor ladder as Phase 2** — do NOT attempt a direct FORCE up/downwind numeric comparison in Phase 3.
- **D-02:** The ladder has three rungs: (a) a **homogeneous-limit exactness anchor** — with `|ξ|` below the clamp threshold the refracted machinery must reproduce the Phase-2 straight-ray results **bit-for-bit** (the circular-ray constructor fills `RayVars`/`RayPair` behind identical fields; the homogeneous path must be numerically indistinguishable); (b) a **committed scipy oracle** (extend `tools/nord2000_oracle/`) pinning ξ, Δτ, the CalcEqSSP equivalent-linear collapse, and CalcEqSSPGround — regenerated by `gen_*.py`, **no Python at test time**; (c) **property tests for direction** — downward refraction → level gain, upward refraction → shadow-zone loss, monotonic where the physics demands it.
- **D-03:** FORCE up/downwind (wind `u`, temperature-gradient `dt/dz`) cases **stay capability-gated** — their `requires:` list shrinks (drop `refraction`) but retains `emission-model`, so they remain `Skipped`, never a false `Pass`, until the Phase-4 road-emission model lands. The `Capability::Refraction` gate (already keyed off nonzero `u`/`dt/dz` in `capability.rs`) is flipped to implemented.
- **D-04:** Honor the **oracle-independence caveat** (CLAUDE.md): the scipy oracle reimplements the same equation transcriptions as the engine, so it cross-checks *implementation*, not *spec reading*. Verify the transcribed ξ/Δτ/A-B/shadow-zone equations against the **AV 1106/07 rev.4 PDF page images** (`refs/AV1106-07-rev4.pdf`), not a research summary — multiple transcription errors were caught this way in Phases 1–2.

### Weather-input routes
- **D-05:** Build **all three routes now**, per the roadmap as-written (MET-05 Route 1 + MET-06 Route 3 both mapped to Phase 3) — no deferral to v2.
- **D-06:** **Route 2** (FORCE-driven): surface wind `u@zu` / temperature-gradient `dt/dz` → A/B coefficients; this drives the up/downwind straight-road cases. Reads the meteorology fields already parsed into the harness case model (`t0`, `u`, `φ`, `su`, `dt/dz`, `sdt/dz`, `Cv²`, `Ct²`, roughness).
- **D-07:** **Route 1** (weather classes): a table of (A,B) pairs with occurrence probabilities → energy-weighted L_den-style combination; validation target is the **Yearly Average** workbook (`Met. statistics` sheet).
- **D-08:** **Route 3** (Monin–Obukhov): reconstruct u(z), T(z) from surface met (cloud cover as stability proxy) and **least-squares fit A, B, C**. Since **no weather API exists yet** (that's v2 METX), Route 3 is validated against **synthetic/derived surface-met inputs + a committed oracle fixture** rather than live data. Keep the LSQ A/B/C-fit code behind a clean boundary so the v2 weather-import path plugs in without a rewrite.

### Shadow zone
- **D-09:** Implement the **full AV 1106/07 upward-refraction / shadow-zone attenuation model** as specified — success criterion 1 explicitly requires shadow-zone cases to match, so it is in scope, not optional. Do NOT rely solely on the equivalent-linear ray model's natural behavior with a documented limit.

### F_τ turbulence coherence
- **D-10:** F_τ enters as an **additional factor in the already-built F chain** `F = Ff·FΔν·Fc·Fr·Fs`, through the **pre-built FΔν seam** (`CoherenceInputs::f_delta_nu`, currently `1.0` from Phase 2 callers) — Phase 3 drops in Δτ⁺ under the A⁺ profile without touching call sites. It feeds the **two-channel readout** unchanged: `L = 10·lg(|coherent Σ (phase intact)|² + P_incoh)`; F_τ never overwrites phase in `H_coh`.
- **D-11:** Cv²/Ct² come from the **FORCE-case fields** (`Cv2`, `Ct2`), with **Nord2000 defaults** when a case omits them. Validate by **property tests** (F_τ < 1 under turbulence, monotonic, correct blend direction) — the same approach Phase 2 used for Fc/Fr; do NOT pin F_τ against a fixed-value scipy oracle (its exact constants are AV Assumptions, inert on zero-turbulence cases and untrustworthy to pin).

### Load-bearing conventions carried forward (do not violate)
- **D-12:** **Complex/phase contract** — the canonical transfer is `H_coh: Complex<f64>` with phase preserved through the refraction operator too; turbulence-decorrelated energy stays in the separate real `P_incoh` channel added only at readout. `F→1 ⇒ P_incoh→0`.
- **D-13:** **Time-convention quarantine** — refraction math is implemented **verbatim in Nord2000's e^{−jωt} convention inside `propagation/`**; the single `.conj()` boundary stays at `transfer::nord_ratio_to_transfer` (grep gate: zero `.conj()` in `propagation/`). Write conjugation as explicit `Complex::new(re, -im)`.
- **D-14:** **Frequency framework** — 105-point 1/12-octave grid; **compare by BAND INDEX, never nominal frequency**. CalcEqSSPGround's f_L/f_H interpolation is evaluated natively at these points.
- **D-15:** **Engine dependency quarantine** — `envi-engine` stays `ndarray + num-complex + thiserror` only (`cargo tree` gate); all new I/O (oracle fixtures, weather-class tables, Route-3 met inputs) lives in `envi-harness`.
- **D-16:** **Numerics house rules** — `z₀` clamped ≥ 0.001 m; ξ singularity / homogeneous-atmosphere singularity clamped; Δτ via the cancellation-free `ΔR = (R₂²−R₁²)/(R₂+R₁)` identity, extended to the circular-ray (refracted) case.

### Claude's Discretion
- Module layout under `propagation/` for the new refraction code (e.g. `refraction/` submodule with profile / eq-linear-collapse / circular-ray / shadow-zone units), the exact ξ clamp threshold value, and the Route-3 LSQ solver choice — planner/researcher decide, subject to the conventions above.
- Exact synthetic surface-met fixtures and oracle-fixture granularity for Route 3.
</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **`RayVars`/`RayPair` (rays.rs):** explicitly designed as the Phase-3 seam — the circular-ray (refracted) constructor fills identical fields (`tau`, `r`, `psi_g`, `r1`, `r2`, `dtau`). The homogeneous-limit exactness anchor (D-02) rides on this: `|ξ|<clamp` must reproduce these exact values.
- **`CoherenceInputs::f_delta_nu` (coherence.rs):** the FΔν injection seam — Phase 2 passes `1.0`; Phase 3 supplies Δτ⁺ under the A⁺ profile with no call-site change (D-10). `Fs` is a documented `=1.0` stub to activate for scattering-zone/shadow behavior.
- **Two-channel transfer (transfer.rs):** `H_coh` complex + `P_incoh` real readout already frozen — refraction multiplies `H_coh`, turbulence adds to `P_incoh`; no new plumbing needed for the readout law.
- **Cancellation-free ΔR identity (rays.rs):** extend from the flat/image-point form to the circular-ray reflected case (D-16).
- **scipy oracle harness (tools/nord2000_oracle):** established pattern — add refraction `gen_*.py` scripts producing committed fixtures; Rust tests run without Python.

### Established Patterns
- **Capability gating (fail-soft):** cases report `Skipped(requires: …)` when a capability is unimplemented — never a false Pass. Phase 3 flips `Refraction` and shrinks the FORCE `requires:` list (D-03).
- **Compare-by-band-index:** never by nominal frequency (D-14) — a recurring pitfall guarded across phases.
- **Nord2000-native convention quarantine:** all new refraction math stays e^{−jωt} inside `propagation/`; explicit `Complex::new(re,-im)` for conjugation, single `.conj()` at the transfer boundary (D-13). A grep gate enforces zero `.conj()` in `propagation/`.
- **Interface-block extensions documented, not silent:** when a new input is needed (as `d_m` was for Fc), add it explicitly and document the deviation.

### Integration Points
- Refraction ray variables → `terrain_effect` Eq. 332 composition (the refracted `RayPair` replaces the straight-ray one on non-homogeneous cases).
- Per-azimuth A derivation consumes the existing `azimuth_deg` (geometry.rs, clockwise-from-north, documented for the `A ∝ u·cos(az−φ_u)` wind-projection consumer).
- Weather routes (harness) produce the (A, B, C) profile that the engine's refraction entry point consumes; Route 1 combination and Route 3 LSQ fit live in `envi-harness` (I/O quarantine, D-15).

</code_context>

<specifics>
## Specific Ideas

- The **homogeneous-limit bit-for-bit anchor** (D-02) is the load-bearing safety net: it proves the refracted machinery is a strict generalization of the Phase-2 straight-ray path, so no regression is introduced on the already-green ground/diffraction cases when `|ξ|` is below the clamp.
- Route 3's LSQ A/B/C fit boundary should be shaped so the **v2 METX weather-import** path (Open-Meteo/ERA5) can supply the surface met later with no rewrite — the fit consumes reconstructed u(z)/T(z), agnostic to whether they came from a FORCE fixture or a live API.

</specifics>
