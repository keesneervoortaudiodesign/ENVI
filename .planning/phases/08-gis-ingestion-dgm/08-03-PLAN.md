<objective>
Add the ONLY new server-side surface this phase: an allowlisted byte/range relay on `envi-service` so
the browser can fetch the two CORS-blocked terrain/landcover sources (Copernicus GLO-30 S3, ESA
WorldCover S3). Direct-fetch sources (PDOK AHN, Overpass) do NOT use it.

Purpose: D-02 — direct cross-origin fetch first, proxy only where CORS blocks. The relay is bytes-only
so "calculations on the local machine" holds, and it is SSRF-proof by construction (hardcoded host +
prefix allowlist, GET-only, no user URLs).
Output: `proxy.rs` relay + route registration + error mapping + rustls HTTP client in state +
`contract_proxy.rs` (with SSRF rejection cases).
