<objective>
Create the first WASM crate in the repo: a thin `envi-gis-wasm` cdylib exposing the `envi-gis` core to
the browser, wire the wasm-bindgen build into the existing Vite pipeline with a version-locked CLI, and
route the WASM boundary DTOs through the same ts-rs generate-and-commit no-drift mechanism used for the
HTTP wire.

Purpose: this is the boundary that lets TypeScript call the pure-Rust ingestion core; getting the
CLI-version lockstep and the generated-types discipline right here prevents the classic wasm-bindgen and
type-drift failures.
Output: `envi-gis-wasm` crate + boundary DTOs + Vite/wasm build wiring + pinned CLI (Wave-0) +
generated committed TS + no-drift test extension.
</objective>
