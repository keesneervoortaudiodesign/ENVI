# Phase 8 Plan 8: Offline Playwright Import Journey + DATA-04 Network-Off Replay Summary

**Two offline Playwright specs prove the browser GIS pipeline end-to-end against tiny committed fixtures: a viewport import that lands editable terrain/land-cover/building features (DGM re-triangulate, SC5 attribution, SC3 impedance overlay, D-07 partial-failure retry), and the load-bearing DATA-04 guarantee — after import, with the network inverted, the compute path reads terrain/land-cover only from OPFS (zero GIS egress) with a negative guard proving it — all ending with the offline-clean `toEqual([])` collector.**

## Accomplishments

- **Real offline import journey.** `import.spec.ts` drives the REAL import orchestrator through the `window.__enviTest` bridge over committed fixtures: terrain (100 `elevation_point`s from an AHN RD-New tif via PDOK), land cover (4 `ground_zone`s from a WorldCover u8 tif via the byte proxy), and buildings (2 `building`s from Overpass) all commit as editable features; the DGM re-triangulates (SC1), SC5 attribution renders for all three sources, and the SC3 impedance overlay toggles on — with a D-07 partial-failure branch (Overpass 429 errors only buildings, retry recovers, siblings untouched).
- **DATA-04 network-off proof.** `import-offline-replay.spec.ts` populates OPFS via an online import, inverts the network (every GIS route re-registered to record + abort), and re-drives the compute read against a reset scene: it asserts BOTH honest properties — zero GIS egress on the OPFS read path (dedicated egress collector + the boot offline-guard collector both `[]`, layers still commit) AND a real OPFS read with a negative guard (evicting the cached terrain tile makes the same path miss OPFS and attempt a now-blocked fetch that fails).
- **Unblocked the import path (Rule 1).** Discovered and fixed the 08-07 WASM boundary bug that made every import layer error (`res.features.features is not iterable`): `serde_wasm_bindgen` serialized GeoJSON payload maps as JS `Map`s. Now serialized as plain objects; committed web/dist rebuilt so the fix ships.
- **Committed fixtures + generator.** A dev-time `gen_e2e_fixtures.py` (rasterio/GDAL + pyproj, never a test dependency) emits ~6 KiB of fixtures aligned to a committed WGS84 viewport inside kaartblad M_25GN1, with a `viewport.json` single source of truth. Validated against the real Rust decode path before writing the specs.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] serde_wasm_bindgen serialized GeoJSON payloads as JS Maps**
- **Found during:** Task 1 (first import run — every layer errored `res.features.features is not iterable`).
- **Issue:** the 08-07 import compute path was never exercised end-to-end (E2E deferred to 08-08). `serde_wasm_bindgen::to_value` serializes every `serde_json::Value::Object` inside the `features` payloads as a JS `Map`, so the TS side read `undefined` and all three layers failed.
- **Fix:** `to_js` now uses `Serializer::new().serialize_maps_as_objects(true)`; rebuilt the wasm + committed web/dist.
- **Files modified:** `crates/envi-gis-wasm/src/lib.rs`, `web/dist/*`, `web/src/generated/wasm/*` (git-ignored artifact).
- **Commit:** `3e880a0`

**2. [Rule 3 - Blocking] Extended the DEV test bridge + added `opfs.removeTile`**
- **Found during:** Task 1/2 — the bridge had no way to drive `runImport`, toggle layers/overlay, read import status, or evict an OPFS tile (the negative guard).
- **Fix:** added DEV-only bridge methods (`runImport`, `setImportLayerEnabled`, `retryImportLayer`, `importState`, `toggleImpedanceOverlay`, `sceneEpoch`, `cachedTile`, `evictTile`) + a production `opfs.removeTile` cache-eviction helper.
- **Files modified:** `web/src/testBridge.ts`, `web/src/import/opfs.ts`
- **Commit:** `92fe4fc`

**3. [Rule 3 - Blocking] Capped Playwright workers at 4**
- **Found during:** first full-suite run — 9 tests (mine + pre-existing) timed out at 30 s under the default worker count.
- **Issue:** the offline suite renders MapLibre via software WebGL (swiftshader, CPU-bound); the default (~half the 22 cores) over-subscribes and starves the contexts.
- **Fix:** `workers: 4` in `playwright.config.ts` — the full suite (17 tests) goes green in ~59 s.
- **Commit:** `92fe4fc`
