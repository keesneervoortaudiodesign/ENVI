|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Integrity (domain-critical) | recondition serving stale/mismatched tensor results | mitigate | The 409 content-hash gate IS the mitigation (SC4/D-07) — contract-tested with expected/got in the body; identity excludes conditioning so the gate cannot be gamed via readout params |
| T-06-04-02 | DoS | SSE connection exhaustion (many EventSource tabs) | accept | Localhost single-user; watch receivers are cheap (no per-subscriber queues); 15 s keep-alive; documented accepted risk per 06-RESEARCH Security Domain — Phase 10 revisits with the job semaphore |
| T-06-04-03 | DoS | unbounded job-registry growth / thread-per-job leak on cancel-spam | accept (bounded mitigation) | Cancel is idempotent via token; workers exit on token check; submission is user-driven on localhost; registry eviction + semaphore bound are Phase-10 scope (documented in jobs.rs header) |
| T-06-04-04 | DoS | malformed recondition bodies (oversized filter arrays, junk hashes) | mitigate | deny_unknown_fields + filter length-105 validation + hash compared as opaque strings (no parsing of attacker-controlled hex); axum default body limit retained |
| T-06-04-05 | Repudiation / Spoofing (of results) | stub output mistaken for real acoustics | mitigate | stub: true provenance in every canned response and manifest; test-asserted (honest-stubs rule) |
| T-06-04-06 | Tampering | CPU work starving the async runtime (degraded cancel/SSE latency as a availability failure) | mitigate | Dedicated std::thread worker (Anti-Pattern 5) — grep-gated; watch send_replace never blocks on slow consumers |
| T-06-SC | Tampering (supply chain) | cargo installs (tokio-util, tokio-stream) | mitigate | All packages OK in the 06-RESEARCH Package Legitimacy Audit; versions pinned per audit |
