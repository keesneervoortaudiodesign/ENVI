<objective>
Create the **`envi-store`** crate: the serde DTO mirror (D-05 — serde lives HERE, never in the engine)
plus the project-as-folder persistence layer (D-04 — flat files, not SQLite: git-diffable,
human-inspectable, copyable) and the frozen tensor-identity hash (D-07). This is the storage half of
SC1 and the wire-DTO half of SC4/SVC-07; plan 06-03's HTTP handlers are thin delegates onto this crate.

Layout per D-04: `projects/<uuid>/project.json` (metadata + settings + the pinned UTM CRS from
plan 06-01's `ProjectCrs`) + `scene.geojson` (RFC 7946 FeatureCollection persisted **in WGS84** —
06-RESEARCH assumption A3, adopted: the file stays valid GeoJSON for external viewers; reprojection to
engine SceneXY space happens on conversion, through the one envi-geo seam) +
`calc/<calc_id>/manifest.json` (dims `[S,R,105]`, receiver-axis chunk layout, content hashes, honest
`stub` provenance flag; `tensor/` + `pincoh/` dirs reserved empty for Phases 9/10).

CRUD contract per D-06: create / open / save (autosave = every mutation persists immediately and
atomically; the server is authoritative, no dirty state) / duplicate (excluding `calc/`) / delete +
reopen-last via `projects/.envi-state.json`. SQLite remains the documented upgrade path only (D-05);
the DTO mirror keeps that swap mechanical.

Purpose: SC1's restart-survival and the SC4 hash-identity contract are only as good as this layer;
everything here is a non-retrofittable contract the UI (Phase 7) and real compute (Phases 9-11) bind to.
Output: the `envi-store` crate — dto, geojson, project_dir, manifest, hash modules with full unit tests.
</objective>
