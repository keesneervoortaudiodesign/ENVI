<objective>
Turn the engine's frozen forward contract (`TransferTensor = Array3<Complex<f64>>` indexed `[sub_source, receiver, freq]`, already typedef'd in `transfer.rs`) into a real, streamable artifact with two readout laws and interactive conditioning.

**Phase-4 slice (MVP framing):** after this plan a downstream caller can, for the first time, ask the engine to fill a dense complex transfer tensor for an arbitrary set of (sub-source, receiver) pairs and then recompute receiver spectra from source conditioning alone — the interactive-recalc payoff — without re-running any propagation.

Purpose: OUT-01..06 are the load-bearing output contract of the whole milestone and the Phase-10 service hard-gates on these types being **public engine API**, not harness-private. Build them once, correctly, as pure math.
Output: `tensor.rs` (TensorPair + TensorSink + InMemorySink + coherent MAC + incoherent Annex-A readout), `solver.rs` (SolveJob seam + chunked solve loop), new typed `PropagationError`/sink error variants, and two harness integration tests (budget + MAC bit-identity).

This plan adds ZERO dependencies and touches NO `propagation/` math (only new error variants in `propagation/mod.rs`).
</objective>
